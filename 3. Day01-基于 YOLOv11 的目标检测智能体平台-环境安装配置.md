## 一、基础环境安装

### 1.1 Git 与 GitHub 版本控制

Git 是项目协作开发的版本控制工具，所有代码通过 Git + GitHub 进行管理和协作。

#### 1.1.1 安装 Git

**Windows：**

1. 访问 Git 官网下载页面：https://git-scm.com/download/win
2. 下载 **64-bit Git for Windows Setup**
3. 运行安装程序，保持默认选项即可，以下几点注意：
   - **Adjusting your PATH environment** → 选择 `Git from the command line and also from 3rd-party software`（推荐）
   - **Choosing the default editor** → 可选择 VS Code 作为默认编辑器
   - **Configuring the line ending conversions** → 选择 `Checkout Windows-style, commit Unix-style line endings`
4. 安装完成后，右键菜单会出现 "Git Bash Here" 选项

**macOS：**

```bash
# 方法 1：使用 Homebrew（推荐）
brew install git

# 方法 2：使用 Xcode Command Line Tools
xcode-select --install
```

**Linux（Ubuntu/Debian）：**

```bash
sudo apt update
sudo apt install git -y
```

**验证安装：**

```bash
git --version
# 期望输出：git version 2.x.x
```

#### 1.1.2 Git 基础配置

安装完成后，需要配置你的身份信息，这会出现在每次 Git 提交记录中：

```bash
# 设置用户名（建议使用真实姓名拼音或英文名）
git config --global user.name "你的名字"

# 设置邮箱（建议与 GitHub 注册邮箱一致）
git config --global user.email "your.email@example.com"

# 设置默认分支名称为 main
git config --global init.defaultBranch main

# 查看当前配置
git config --list
```

#### 1.1.3 GitHub 账号与 SSH 密钥配置

**第一步：注册 GitHub 账号**

访问 https://github.com/signup ，使用邮箱注册并完成验证。

**第二步：生成 SSH 密钥**

```bash
# 检查是否已有 SSH 密钥
ls -al ~/.ssh

# 如果没有 id_ed25519.pub 文件，则生成新的 SSH 密钥
ssh-keygen -t ed25519 -C "your.email@example.com"
# 一路回车即可（也可以设置密码短语增加安全性）
```

**第三步：将公钥添加到 GitHub**

```bash
# Windows（Git Bash 中执行）
cat ~/.ssh/id_ed25519.pub

# macOS
pbcopy < ~/.ssh/id_ed25519.pub    # 直接复制到剪贴板

# Linux
xclip -sel copy < ~/.ssh/id_ed25519.pub   # 需要安装 xclip
```

然后：

1. 打开 https://github.com/settings/keys
2. 点击 **New SSH key**
3. Title 填写一个描述性名称（如 "我的笔记本"）
4. Key type 选择 **Authentication Key**
5. 将刚才复制的公钥内容粘贴到 Key 框中
6. 点击 **Add SSH key**

**第四步：验证连接**

```bash
ssh -T git@github.com
# 如果看到：Hi xxx! You've successfully authenticated, but GitHub does not provide shell access.
# 说明配置成功
```

> **首次连接提示**：会询问 `Are you sure you want to continue connecting (yes/no)?`，输入 `yes` 回车即可。

#### 1.1.4 Git 常用命令速查

```bash
# 克隆远程仓库到本地
# HTTPS方式（推荐）
https://github.com/your-username/rsod-web-platform.git

# SSH方式（需要配置SSH密钥，参见上一步）
git@github.com:your-username/rsod-web-platform.git

# 查看当前状态（最常用的命令）
git status

# 创建并切换到新分支
git checkout -b feature/your-feature-name

# 添加文件到暂存区
git add .                    # 添加所有修改
git add backend/main.py      # 添加指定文件

# 提交更改（写清楚本次修改的内容）
git commit -m "feat: 添加目标检测API接口"

# 推送本地分支到远程
git push origin feature/your-feature-name

# 拉取远程仓库最新代码
git pull origin main

# 查看所有分支
git branch -a

# 查看提交历史
git log --oneline
```

**分支命名建议：**

| 前缀 | 用途 | 示例 |
|------|------|------|
| `feature/` | 新功能 | `feature/detection-api` |
| `fix/` | Bug 修复 | `fix/login-error` |
| `docs/` | 文档更新 | `docs/api-guide` |
| `refactor/` | 代码重构 | `refactor/agent-router` |

### 1.2 Python 安装与虚拟环境

Python 是本项目后端开发、模型推理的核心语言。本项目要求 **Python 3.10 及以上版本**。

#### 1.2.1 安装 Python

**Windows：**

1. 访问 Python 官网下载页面：https://www.python.org/downloads/
2. 下载 Python 3.10.x 或 3.11.x 版本（推荐 3.10.20 或 3.11.9）
3. **⚠️ 重要：安装时务必勾选 "Add Python to PATH"**
4. 点击 "Install Now" 完成安装

**macOS：**

```bash
# 方法 1：使用 Homebrew（推荐）
brew install python@3.10

# 方法 2：使用 pyenv 管理多版本（适合需要多个 Python 版本的场景）
brew install pyenv
pyenv install 3.10.20
pyenv global 3.10.20
```

**Linux（Ubuntu/Debian）：**

```bash
sudo apt update
sudo apt install python3.10 python3.10-venv python3-pip -y
```

**验证安装：**

```bash
python --version      # Windows
python3 --version     # macOS / Linux
# 期望输出：Python 3.10.x 或 Python 3.11.x

pip --version          # Windows
pip3 --version         # macOS / Linux
# 期望输出：pip 23.x.x 或更高
```

#### 1.2.2 创建与管理虚拟环境

虚拟环境用于隔离项目依赖，避免不同项目之间的包版本冲突。**每个项目都应该使用独立的虚拟环境。**

**创建虚拟环境：**

```bash
# 进入项目根目录
cd rsod-agent-platform/backend

# 创建虚拟环境（推荐命名为 .venv）
python -m venv .venv        # Windows
python3.10 -m venv .venv       # macOS / Linux
```

**激活虚拟环境：**

```bash
# Windows（CMD）
.venv\Scripts\activate

# Windows（PowerShell）
.venv\Scripts\Activate.ps1

# macOS / Linux
source .venv/bin/activate

#查看当前环境 python 路径
which python
```

激活成功后，终端提示符前会出现 `(.venv)` 标识：

```bash
(.venv) C:\Users\xxx\rsod-agent-platform>     # Windows
(.venv) user@hostname:~/rsod-agent-platform$   # macOS / Linux
```

**退出虚拟环境：**

```bash
deactivate
```

> **⚠️ 注意**：后续所有 `pip install` 操作都必须在虚拟环境激活状态下执行。

#### 1.2.3 配置 pip 镜像源（国内用户）

由于网络原因，国内直接访问 PyPI 速度较慢，建议配置阿里云镜像源：

```bash
# 临时使用（单次安装）
pip install <package> -i https://mirrors.aliyun.com/pypi/simple

# 永久配置（推荐）
pip config set global.index-url https://mirrors.aliyun.com/pypi/simple
pip config set global.trusted-host mirrors.aliyun.com
```

#### 3.1.1 Windows 安装

**前提条件：**

- Windows 10（版本 2004 及以上）或 Windows 11
- 需要启用 WSL 2（Windows Subsystem for Linux 2）

**安装步骤：**

1. **启用 WSL 2**（如果尚未启用）：

   以管理员身份打开 PowerShell，执行：

   ```powershell
   # 启用 WSL
   dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart

   # 启用虚拟机平台
   dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart

   # 重启电脑
   shutdown /r /t 0
   ```

   重启后，继续执行：

   ```powershell
   # 设置 WSL 2 为默认版本
   wsl --set-default-version 2

   # 安装默认的 Ubuntu 发行版
   wsl --install -d Ubuntu
   ```

2. **下载并安装 Docker Desktop**：
   - 访问 https://www.docker.com/products/docker-desktop/
   - 下载 **Windows - AMD64** 版本
   - 运行安装程序，勾选 **Use WSL 2 instead of Hyper-V**
   - 安装完成后重启电脑

3. **启动 Docker Desktop**：
   - 从开始菜单启动 Docker Desktop
   - 等待左下角状态变为 **绿色（Engine running）**
   - 首次启动可能需要等待 1-2 分钟

#### 3.1.2 macOS 安装

1. 访问 https://www.docker.com/products/docker-desktop/

      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U rsod_admin -d rsod_agent"]
      interval: 5s
      timeout: 5s
      retries: 5

  # ── Redis ──────────────────────────────────────────
  redis:
    image: redis:7-alpine
    container_name: rsod-agent-redis
    restart: unless-stopped
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 5s
      retries: 5

  minio:
    image: minio/minio:latest

      - minio_data:/data
    command: server /data --console-address ":9001"

### Q1：Windows 下 `python` 命令提示 "不是内部或外部命令"

**原因**：安装 Python 时未勾选 "Add Python to PATH"。

**解决**：

1. 打开 Windows 设置 → 搜索"环境变量"
2. 点击"编辑系统环境变量" → "环境变量"
3. 在"Path"中添加 Python 安装路径，例如：
   ```
   C:\Users\你的用户名\AppData\Local\Programs\Python\Python310\
   C:\Users\你的用户名\AppData\Local\Programs\Python\Python310\Scripts\
   ```
4. 重启终端

或者直接重新运行 Python 安装程序，勾选 "Add Python to PATH"。

### Q5：Docker Compose 启动时端口被占用

例如 `Bind for 0.0.0.0:5432 failed: port is already allocated`。

**解决**：

```bash
# 查看哪个进程占用了端口
# Windows:
netstat -ano | findstr :5432

# macOS / Linux:
lsof -i :5432

# 方案 1：停止占用端口的程序
# 方案 2：修改 docker-compose.yml 中的端口映射，如 5432:5432 改为 15432:5432
```

### Q6：npm install 报错 "EACCES permission denied"（macOS / Linux）

**解决**：不要使用 `sudo npm install`，而是修复 npm 目录权限：

```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.zshrc   # macOS
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc   # Linux
source ~/.zshrc   # 或 source ~/.bashrc
```

### Q7：`ultralytics` 安装失败或 import 报错

**常见原因**：系统缺少编译工具或依赖库。

**解决**：

```bash
# Windows：安装 Visual C++ Build Tools
# 下载：https://visualstudio.microsoft.com/visual-cpp-build-tools/
# 安装时勾选 "C++ build tools"

# macOS：安装 Xcode Command Line Tools
xcode-select --install

# Linux：安装编译工具
sudo apt install build-essential python3-dev -y
```

### Q8：Ollama 模型下载很慢或失败（备选，可不下载）

Ollama 需要下载 LLM 模型文件（通常几个 GB），国内网络可能较慢。

**解决方案**：

1. 暂时不启动 Ollama 服务，使用 OpenAI API 作为替代
2. 在 `.env` 中设置 `USE_LOCAL_LLM=false`
3. 在 `docker-compose.yml` 中注释掉 `ollama` 服务

```yaml
  # ollama:
  #   image: ollama/ollama:latest
  #   ...
```
