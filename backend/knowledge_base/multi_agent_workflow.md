# 多智能体工作流知识

本项目的多智能体不是让多个模型都去直接看图片，而是把交通检测任务拆成可解释的专业步骤。Supervisor Agent 负责判断用户意图和调度流程；Detection Agent 调用 YOLO 模型检测图片、ZIP、视频或摄像头帧；Analysis/Risk Agent 根据结构化检测结果、可见度指标、车辆密度和大车比例计算风险等级；QA/RAG Agent 从本地知识库解释 IoU、mAP、YOLO、视频去重和雨雾预警规则；Summarize Agent 汇总成用户能看懂的回答。

大模型只应接收结构化摘要，例如：车辆类别统计、去重车辆数、采样帧数、平均置信度、可见度等级、风险等级、预警原因和管理建议。不要把整张图片、视频、base64 图像或完整边界框列表直接发送给大模型，否则会造成 token 消耗过高，也会暴露不必要的数据。

用户上传图片并提问“有多少车”时，推荐流程是 Supervisor -> Detection -> Summarize。用户同时问“什么是 IoU”时，推荐流程是 Supervisor -> Detection + QA/RAG 并行 -> Summarize。用户只问项目知识时，不需要调用 YOLO，只调用 QA/RAG 或普通对话即可。

回答应避免把后台工具结果原样堆给用户。比如用户问车辆数量时，只需要返回类别统计和结论；用户没有明确询问平台账号时，不应该返回用户列表；用户没有询问历史记录时，不应该混入历史任务数据。
